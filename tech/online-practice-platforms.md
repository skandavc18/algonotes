I often complain about exam-oriented education, but I did pick up one trick from it: **if you can learn a skill in the form of doing problems, the efficiency and effect are the best**. When learning tech, I often face this dilemma — I know plenty of theory, but some scenarios are hard to simulate, and I lack hands-on practice. If only there were an exercise book with reference answers I could grind through.

So when learning a new tech, I first search for online practice platforms. And it turns out some great folks have built really good ones. Below I introduce three platforms — for practicing Git, SQL, and regular expressions.

### 1. Practicing Git

This is a project called Learning Git Branching, and I recommend it without reservation:

![](https://labuladong.online/algo/images/online/1.png)

As the dialog says, this is by far **the best** Git animated tutorial I've ever seen.

Back in the day, the only Git commands I knew were `add .`, `clone`, `push`, `pull`, `commit`. I had no clue about anything else. To me Git was a downloader and GitHub was a free file host. Whether the commands actually achieved what I wanted was up to luck. Version control? I had no idea, and was too lazy to read the messy docs.

This site doesn't give you tedious examples about modifying files. Instead it abstracts each `commit` into a tree node, and **uses an animated, level-based format** that lets you freely use Git commands to reach the goal:

![](https://labuladong.online/algo/images/online/2.png)

All Git branches are visualized — type Git commands into the command line on the left and the branches change accordingly. Once you achieve the task goal, you pass the level! The site also tracks how many commands you used. See if you can clear it with the minimum.

![](https://labuladong.online/algo/images/online/3.png)

I originally thought the tutorial only covered local Git repository version management — **then I was pleasantly surprised to find it also has tutorials for remote repositories!**

![](https://labuladong.online/algo/images/online/4.png)

![](https://labuladong.online/algo/images/online/5.png)

It really does feel like playing a game. The difficulty is well-designed and the flow is smooth. Once I started I couldn't stop and finished it in a few hours, haha!

![](https://labuladong.online/algo/images/online/6.png)

Overall, this tutorial is great for beginners and intermediates alike. If you feel your Git skills aren't great and you still rely on luck when typing Git commands, give this tutorial a try — I'm sure it will help you use Git much more fluently.

It's an open-source project. GitHub:

https://github.com/pcottle/learnGitBranching

Tutorial site:

https://learngitbranching.js.org

### 2. Practicing regular expressions

**Regular expressions are an extremely powerful tool**. You could say everything in a computer is characters, and with a pattern-matching tool like regex, operating a computer becomes much more powerful.

I'll recommend two sites — one is a practice platform, the other is a regex testing tool.

The practice platform is RegexOne:

![](https://labuladong.online/algo/images/online/9.png)

It starts with basic tutorials and follows up with common regex problems — validating emails, URLs, phone numbers, extracting key info from logs, and so on.

Once you write a regex that meets the requirements you can move to the next problem. Each problem also has a reference answer that you can view by clicking the `solution` button below:

![](https://labuladong.online/algo/images/online/10.png)

RegexOne URL:

https://regexone.com/

Now the testing tool — a GitHub project called RegExr. Its website:

![](https://labuladong.online/algo/images/online/11.png)

As you can see, after entering text and a regex pattern, **the site formats the regex with nice, easy-to-read styling, automatically searches the text for matches, highlights matched substrings, and shows the strings captured by each group**.

This site pairs nicely with the regex practice platform above — try out expressions here, and once they work, paste them over.

RegExr URL:

https://regexr.com/

### 3. Practicing SQL

This is a site called SQLZOO. The left side lists all the practice content:

![](https://labuladong.online/algo/images/online/7.png)

SQLZOO is a very handy SQL practice platform. The English isn't hard to understand, you can use it directly, but you can also switch to Traditional Chinese — quite friendly.

It covers commonly used SQL commands. Given a requirement, you write a SQL statement to produce the correct query result. **The best part: it not only explains each command in detail, but each topic also has multiple-choice quizzes with auto-grading. Some of the harder problems even have video walkthroughs**:

![](https://labuladong.online/algo/images/online/8.png)

The difficulty is progressive and friendly even to beginners; later problems get genuinely tricky — perfect for those who love mental challenges! LeetCode also has SQL problems, but they're generally harder. I think it's better to grind SQLZOO first to learn the basics, then move on to LeetCode.

Site:

https://sqlzoo.net/
